# 问题1：执行meson setup build 时输出如下报错
![e02fc654f4c545fecf52a888f1bb31b1.png](../../_resources/e02fc654f4c545fecf52a888f1bb31b1.png)

解决：安装缺失的Python模块（推荐）
```
bash
# 在容器内执行以下命令

# 1. 首先安装pip（如果还没安装）
apt-get install -y python3-pip

# 2. 安装pyelftools包
pip3 install pyelftools

# 或者使用apt安装（可能版本较旧，但更稳定）
apt-get install -y python3-pyelftools

# 3. 验证安装
python3 -c "import elftools; print('elftools installed successfully')"
```


# 问题2：./build/helloworld 返回如下报错
![78ae1b81c20b28aa496c1224f3bd5085.png](../../_resources/78ae1b81c20b28aa496c1224f3bd5085.png)

解决：在容器内配置巨页HugePages支持

```
bash
# 在容器内执行以下命令

# 1. 检查当前巨页状态
cat /proc/meminfo | grep -i huge

# 2. 创建巨页目录
mkdir -p /dev/hugepages

# 3. 挂载hugetlbfs
mount -t hugetlbfs nodev /dev/hugepages

# 4. 配置巨页数量（例如配置512个2MB巨页）
echo 512 > /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages

# 5. 验证配置
cat /proc/meminfo | grep -i huge
cat /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages
```

# 问题3：第12步：测试DPDK程序开发,执行 make 时报错
![20f665cf92a2e89e9e077a3098c91a15.png](../../_resources/20f665cf92a2e89e9e077a3098c91a15.png)

解决：使用meson构建（推荐，适用于DPDK 20.11+）
```
# 在容器内执行

# 1. 进入你的应用目录
cd ~/dpdk-workspace/myapp

# 2. 创建meson.build文件
cat > meson.build << 'EOF'
project('hello_dpdk', 'C')

# 查找DPDK依赖
dpdk_dep = dependency('libdpdk', required: true)

# 可执行文件
executable('hello_dpdk', 'hello_dpdk.c',
    dependencies: dpdk_dep,
    install: false)
EOF

# 3. 使用meson构建
meson setup build
cd build
ninja

# 4. 运行程序
./hello_dpdk -l 0 -n 4
```

# 问题4： 重启容器后执行 helloworld 程序报错，提示EAL: 512 hugepages of size 2097152 reserved, but no mounted hugetlbfs found for that size

![95890dedc47938037d0e013c4936d6cd.png](../../_resources/95890dedc47938037d0e013c4936d6cd.png)

解决：因为重启以后，之前 hugepages 配置的挂载目录丢失了，要重新挂载一下，不然就重新创建容器的时候直接配置好挂载卷
这里简单的在容器内重新临时挂载一下
```
mkdir -p /dev/hugepages
mount -t hugetlbfs nodev /dev/hugepages
```
